Calling for test-less package owners

Imagine a world where we don’t use money as currency, but only use our knowledge to help each other and give services to get more knowledge.

Ok, so this sounds a bit hippie and utopia, but we at Chingu decided to try and learn how to test our apps through building tests for oss packages.

Chingu is a community for junior developers, started by Chance Taken, that help each other to learn and progress in the software development world. Although we started in web development, today we have channels focused on machine learning, python, freelancing, digital marketing, cs and of course — test-driven-chingus :)

Lately Jason Boxman offered to start looking for packages that don’t have test suites and build the tests for them. The idea is that we will build the test suite and in the process we will learn, well, the process, and you get a new and shiny test suite.

So we put up the following google form, that you can fill and submit to us and we can start with this amazing project. We target mostly JavaScript packages, but even if your package isn’t written in JS, you are welcome to submit and we will get to it in the future.

Hopefully this project will be a success like many past projects, and we will have this knowledge to use in our jobs or to help other newer chingus.
